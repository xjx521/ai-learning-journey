## 一、浏览器开发者工具（F12）

### 打开方式

| 浏览器 | 快捷键 | 菜单路径 |
|--------|--------|----------|
| Chrome | `F12` 或 `Ctrl+Shift+I` | 右键 → 检查（Inspect） |
| Edge | `F12` 或 `Ctrl+Shift+I` | 右键 → 检查 |
| Firefox | `F12` 或 `Ctrl+Shift+I` | 右键 → 检查元素 |

### Network（网络）面板核心功能

```
打开 F12 → 切到 Network 标签 → 刷新页面
```

你会看到页面上**每一个**网络请求都列出来了：
- HTML 页面本身
- CSS 样式文件
- JS 脚本文件
- 图片（jpg/png/svg）
- API 请求（XHR/Fetch）
- 字体文件（woff2）

#### 过滤器（常用）

| 过滤按钮 | 只看什么 | 什么时候用 |
|----------|---------|-----------|
| **All** | 全部请求 | 默认视图 |
| **Fetch/XHR** | API 请求 | 🔥 最常用！找接口数据 |
| **Doc** | HTML 文档 | 看页面结构 |
| **CSS/JS** | 样式和脚本 | 调试前端资源 |
| **Img** | 图片 | 检查图片加载 |

#### 单个请求的详情面板

点击任意请求，右侧会弹出详情：

| 标签页 | 内容 | 重点关注 |
|--------|------|---------|
| **Headers** | 请求头 + 响应头 + 状态码 | ⭐ 必看 |
| **Payload / Request** | 请求体（POST 时） | 看你发了什么数据 |
| **Preview** | 响应内容（格式化后） | ⭐ 看 API 返回的 JSON |
| **Response** | 原始响应内容 | 和 Preview 类似 |
| **Timing** | 各阶段耗时 | 性能优化时用 |
| **Cookies** | 请求带的 Cookie | 调试登录状态 |

#### Headers 面板详解

```
General（概要）
├── Request URL: 完整请求地址
├── Request Method: GET/POST/...
├── Status Code: 200 OK ✅ / 404 Not Found ❌
└── Remote Address: 服务器 IP

Response Headers（服务器返回的头）
├── content-type: application/json     ← 数据格式
├── cache-control: no-cache            ← 缓存策略
├── set-cookie: session=abc; Path=/    ← 设置 Cookie
└── access-control-allow-origin: *     ← CORS 跨域

Request Headers（你发出去的头）
├── accept: application/json
├── authorization: Bearer xxx
├── cookie: session=abc
├── user-agent: Mozilla/5.0 ...
└── content-type: application/json
```

## 二、实战任务：用 F12 观察真实网站

### 任务 1：观察百度搜索的 API 请求

```
步骤：
1. 打开 https://www.baidu.com
2. F12 → Network → 点击 "Fetch/XHR" 过滤
3. 在搜索框输入 "Python" （先别按回车！）
4. 观察 Network 面板新增了什么请求
5. 点击那个请求，查看：
   - Headers 标签：请求 URL 是什么？
   - Payload 标签：你输入的关键词怎么传的？
   - Preview 标签：百度返回了什么数据？
```

**你会发现**：搜索建议是通过 API 动态获取的，每次输入都会发一个 GET 请求。

### 任务 2：观察 GitHub 登录请求

```
步骤：
1. 打开 https://github.com/login
2. F12 → Network → 勾选 "Preserve log"（保留日志）
3. 输入假用户名密码，点击登录
4. 找到 POST 请求（通常是 /session）
5. 查看：
   - 状态码是多少？（应该是 200 或 302 或 422）
   - Request Payload 里有没有你的密码？（明文还是加密？）
   - Response 里有什么错误信息？
```

### 任务 3：观察视频网站的流媒体请求

```
步骤：
1. 打开 B 站任意视频
2. F12 → Network → 点击 "Media" 过滤
3. 播放视频
4. 观察：视频是怎么加载的？（一个大文件 vs 多个小分片？）
```

**你会发现**：视频是分片加载的（.m4s 或 .ts 文件），这就是流媒体。

## 三、curl 命令行工具

### 什么是 curl？

curl 是一个**命令行下的 HTTP 客户端**，用纯文字发送和接收 HTTP 请求。

```
浏览器（图形界面）：点点点 → 自动发请求 → 渲染页面
curl（命令行）：敲命令 → 发请求 → 显示原始响应
```

### 基本语法

```bash
# 最简单的 GET 请求
curl https://httpbin.org/get

# 带查询参数的 GET
curl "https://httpbin.org/get?name=test&age=21"
# ⚠️ URL 有 & 时必须加引号！

# 查看完整响应头
curl -v https://httpbin.org/get
# -v = verbose（详细模式）

# 只看响应头（不显示响应体）
curl -I https://httpbin.org/get
# -I = HEAD 请求
```

### 发送 POST 请求

```bash
# 发送 JSON 数据
curl -X POST https://httpbin.org/post \
  -H "Content-Type: application/json" \
  -d '{"name": "张三", "age": 21}'

# 参数说明：
# -X POST        → 指定方法为 POST
# -H "..."       → 添加请求头（可以多个 -H）
# -d '...'       → 请求体数据
```

### 常用选项速查表

| 选项 | 全称 | 作用 | 示例 |
|------|------|------|------|
| `-X` | `--request` | 指定 HTTP 方法 | `curl -X DELETE url` |
| `-H` | `--header` | 添加请求头 | `curl -H "Authorization: Bearer xxx"` |
| `-d` | `--data` | 请求体（POST/PUT） | `curl -d '{"key":"value"}'` |
| `-i` | `--include` | 显示响应头 | `curl -i url` |
| `-I` | `--head` | 只发 HEAD 请求 | `curl -I url` |
| `-v` | `--verbose` | 详细输出 | `curl -v url` |
| `-s` | `--silent` | 静默模式 | `curl -s url` |
| `-o` | `--output` | 保存到文件 | `curl -o file.zip url` |
| `-O` | `--remote-name` | 用原名保存 | `curl -O url/file.zip` |
| `-L` | `--location` | 跟随重定向 | `curl -L url` |

### Windows 注意事项

Windows 的 cmd 和 PowerShell 对引号处理不同：
```bash
# cmd 中用双引号包 JSON 会出问题，建议用 PowerShell
# PowerShell 中单引号包 JSON：
curl -X POST https://httpbin.org/post -H "Content-Type: application/json" -d '{"name":"test"}'

# 或者直接安装 Git Bash，用 bash 语法
```

## 四、Postman / Apifox 图形工具

### Postman vs Apifox

| 对比 | Postman | Apifox |
|------|---------|--------|
| 语言 | 英文 | 中文 |
| 安装 | 需下载客户端 | 客户端 + 网页版 |
| 功能 | API 测试 | API 测试 + 文档 + Mock |
| 推荐 | 国际通用 | 🇨🇳 国内推荐，中文友好 |

### 核心操作（两者通用）

```
1. 新建请求
   - 选择方法（GET/POST/...）
   - 输入 URL

2. 设置请求头
   - Headers 标签 → 添加键值对

3. 设置请求体
   - Body 标签 → 选 raw → 选 JSON
   - 输入 JSON 数据

4. 发送 → 查看响应
   - 状态码
   - 响应时间
   - 响应体（Pretty 格式化 / Raw 原始 / Preview 预览）

5. 保存请求到集合（Collection）
   - 方便以后重新测试
```

### 推荐安装

选一个即可：
- **Apifox**（推荐）：https://apifox.com/ — 中文界面，功能全
- **Postman**：https://www.postman.com/ — 国际通用

## 五、Preserve Log 的重要性

勾选 Network 面板的 **"Preserve log"（保留日志）**：

```
不勾选：页面跳转时，之前的请求记录会清空
勾选后：即使页面跳转，所有请求都保留
```

**什么时候必须勾选？**
- 调试登录流程（登录成功后页面会跳转）
- 调试重定向（301/302）
- 表单提交后页面刷新
