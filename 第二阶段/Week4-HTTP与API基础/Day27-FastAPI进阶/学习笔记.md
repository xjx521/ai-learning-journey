## 一、请求体（Request Body）

### 什么是请求体？

```
GET 请求 → 参数在 URL 里（查询参数）
POST/PUT 请求 → 参数在请求体（Body）里

请求体就像 POST 请求的"包裹"，里面装着要提交的数据。
```

### 用 Pydantic Model 定义请求体

```python
from pydantic import BaseModel
from fastapi import FastAPI

app = FastAPI()

# 1. 定义数据模型（继承 BaseModel）
class ItemCreate(BaseModel):
    name: str
    price: float
    description: str | None = None  # 可选字段
    tags: list[str] = []            # 默认空列表

# 2. 在路由中声明为参数
@app.post("/items")
def create_item(item: ItemCreate):  # item 的类型是 ItemCreate
    # FastAPI 自动：
    # - 从请求体中读取 JSON
    # - 验证数据是否符合模型
    # - 转换为 ItemCreate 对象
    return {
        "name": item.name,
        "price": item.price,
        "message": "创建成功"
    }
```

### 测试请求体

用 curl：
```bash
curl -X POST http://localhost:8000/items \
  -H "Content-Type: application/json" \
  -d '{"name": "Python书", "price": 59.9, "tags": ["编程", "教育"]}'
```

用 /docs：
1. 访问 http://localhost:8000/docs
2. 展开 POST /items
3. 点 "Try it out"
4. 编辑 Request body 中的 JSON
5. 点 "Execute"

### 自动验证

```json
// ✅ 正确
{"name": "书", "price": 59.9}

// ❌ 缺少必填字段 name
{"price": 59.9}
// → 422: field required

// ❌ 类型错误
{"name": "书", "price": "五十块"}
// → 422: value is not a valid float

// ❌ 多余字段（默认忽略）
{"name": "书", "price": 59.9, "color": "red"}
// → 正常处理，color 被忽略（除非配置了 extra="forbid"）
```

**这就是 Pydantic 的威力**：你定义一次模型，验证自动生成。

## 二、Pydantic Model 进阶

### 字段验证

```python
from pydantic import BaseModel, Field, EmailStr
from typing import Optional

class UserCreate(BaseModel):
    # Field() 添加额外验证规则
    username: str = Field(..., min_length=3, max_length=20, description="用户名，3-20字符")
    age: int = Field(..., ge=0, le=150, description="年龄，0-150")
    email: EmailStr = Field(..., description="邮箱地址")
    bio: Optional[str] = Field(default=None, max_length=500, description="个人简介")
    # ... 表示必填

# Field 常用参数：
# ...       → 必填（也可以用 default=... 表示）
# default=X → 默认值
# min_length → 字符串最小长度
# max_length → 字符串最大长度
# ge        → 大于等于 (greater or equal)
# le        → 小于等于 (less or equal)
# gt        → 大于 (greater than)
# lt        → 小于 (less than)
# pattern   → 正则表达式匹配
```

### 嵌套模型

```python
class Address(BaseModel):
    city: str
    street: str
    zipcode: str

class User(BaseModel):
    name: str
    address: Address  # 嵌套另一个模型
    hobbies: list[str]  # 列表

# 对应的 JSON：
# {
#     "name": "张三",
#     "address": {"city": "北京", "street": "长安街", "zipcode": "100000"},
#     "hobbies": ["编程", "读书"]
# }
```

## 三、响应模型（Response Model）

### 为什么需要响应模型？

```python
# 问题：数据库模型通常包含密码，但你不希望返回给客户端
class UserInDB(BaseModel):
    username: str
    email: str
    hashed_password: str  # 不应该返回给客户端！

# 解决：定义单独的响应模型
class UserResponse(BaseModel):
    username: str
    email: str
    # 没有 hashed_password

@app.post("/users", response_model=UserResponse)
def create_user(user: UserInDB):
    # FastAPI 会自动过滤掉 response_model 中没有的字段
    return user  # 即使返回了完整对象，hashed_password 也会被过滤
```

### response_model 的作用

1. **过滤字段**：只返回模型中定义的字段
2. **类型转换**：确保返回值符合模型
3. **文档生成**：在 /docs 中显示响应格式

### 不同的输入和输出模型

```python
class TodoCreate(BaseModel):
    """创建待办事项时需要的数据"""
    title: str
    description: str | None = None

class TodoResponse(BaseModel):
    """返回给客户端的数据（多了 id 和完成状态）"""
    id: int
    title: str
    description: str | None = None
    completed: bool = False

@app.post("/todos", response_model=TodoResponse, status_code=201)
def create_todo(todo: TodoCreate):
    # 模拟存入数据库
    return TodoResponse(
        id=1,
        title=todo.title,
        description=todo.description,
        completed=False
    )
```

## 四、路径参数 + 查询参数 + 请求体混合

```python
class ItemUpdate(BaseModel):
    name: str | None = None
    price: float | None = None

@app.put("/items/{item_id}")
def update_item(
    item_id: int,               # 路径参数
    item: ItemUpdate,           # 请求体
    notify: bool = False        # 查询参数
):
    result = {"item_id": item_id, "update": item}
    if notify:
        result["notification"] = "已发送通知"
    return result

# PUT /items/123?notify=true
# Body: {"name": "新书", "price": 49.9}
```

FastAPI 的判断规则：
- 在路径 `{xxx}` 中 → 路径参数
- 类型是 Pydantic Model → 请求体
- 其他简单类型 → 查询参数

## 五、错误处理

### HTTPException

```python
from fastapi import HTTPException

items_db = {1: "苹果", 2: "香蕉"}

@app.get("/items/{item_id}")
def get_item(item_id: int):
    if item_id not in items_db:
        raise HTTPException(
            status_code=404,
            detail=f"商品 {item_id} 不存在"
        )
    return {"item": items_db[item_id]}

# GET /items/999 → 404 {"detail": "商品 999 不存在"}
```

### 自定义错误处理

```python
from fastapi import Request
from fastapi.responses import JSONResponse

@app.exception_handler(ValueError)
async def value_error_handler(request: Request, exc: ValueError):
    return JSONResponse(
        status_code=400,
        content={"message": f"值错误：{str(exc)}"}
    )
```

## 六、数据验证实战

### 用 Pydantic 验证复杂数据

```python
from pydantic import BaseModel, Field, field_validator
from datetime import datetime

class EventCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=100)
    start_time: datetime
    end_time: datetime
    max_attendees: int = Field(..., gt=0, le=10000)

    # 自定义验证器：检查结束时间晚于开始时间
    @field_validator("end_time")
    @classmethod
    def end_must_be_after_start(cls, v, info):
        if "start_time" in info.data and v <= info.data["start_time"]:
            raise ValueError("结束时间必须晚于开始时间")
        return v

# ✅ 正确
# {"title": "会议", "start_time": "2026-07-20T10:00:00", "end_time": "2026-07-20T12:00:00", "max_attendees": 50}

# ❌ end_time 早于 start_time → 422
```
